# KNOWLEDGE_MAP.md

Explains the semantic meaning of the `project/` directory.

This is not workflow.
This is ontology.

---

## project/specs

Defines WHAT the system must do.
This is the living specification.

---

## project/decisions

Defines WHY choices were made.
These override developer preference.

---

## project/issues

Defines WHEN change is required.

No Issue = No Change.

---

## project/tasks

Defines HOW work is decomposed.

---

## Key Principle

Code is disposable.
project knowledge is permanent.
