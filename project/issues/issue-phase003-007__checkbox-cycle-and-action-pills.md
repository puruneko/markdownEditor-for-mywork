# チェックボックス循環＋行ホバーのインライン・アクションピル

## 1. 課題と方針  — 人間が読む

### このissueで解決すること
ステータスやメタの付与が**手打ち中心**で、執筆中・キーボード主体の場面で「文字を打たずにメタを与える」体験が無い（プラン §7.2「自然化」群）。ドラッグ（`issue-phase002-004`）が最適でない場面でも、同じく直接操作で状態・日付・優先度を与えられるようにしたい。

本 issue では低工数・高頻度の2つを実装する: ① エディタ/ビューの `[ ]` クリックで `[ ]→[>]→[x]→[ ]` を循環、② タスク行ホバーで行末に「📅 日付 / ⚑ 優先 / ▸ 状態」のアクションピルを出し、クリックでピッカーを開いてメタを付与。

### 方針
- CM6 のチェックボックス装飾にクリックハンドラを追加し、ステータスマーカーを循環書き戻し。
- CM6 ウィジェット装飾でホバー時に行末ピルを表示、クリックでポップオーバー → 書き戻し。

### 仕様（確定事項：迷ったらこれに従う）
- チェックボックス循環: クリックで **`[ ]→[>]→[x]→[ ]`**（todo→doing→done→todo）。`[!]`/`[-]` はクリック循環には含めない（明示操作はピルの状態メニューから）。
- 状態マーカーの置換は行頭の `[x]` 部分のみ最小パッチ（テキスト本体を触らない）。
- アクションピルはタスク行ホバーで行末に出現。`📅`=日付（schedule/due ピッカー）、`⚑`=優先度、`▸`=状態メニュー（5状態すべて選択可）。
- ピルからの付与は**子メタ行の upsert**（`upsert-meta`）で正規形挿入/更新。
- すべてエディタ装飾レイヤ。本文の書き換えは最小パッチに限定。

### 実装の要点・つまずき
- クリック座標 → 対象行 → AST ノード解決は `nodeLineMap` 逆引き（`issue-phase002-004` の drag 源と同じ解決を流用可）。
- ホバーピルはカーソル/選択と干渉しないよう atomic widget で。連打時の多重書き戻しを抑制。
- 循環書き戻し後に装飾が再構築されてもチラつかないようにする。
