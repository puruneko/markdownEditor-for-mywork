# エディタの折り畳み式フィルタ（行を消さず非表示）

## 1. 課題と方針  — 人間が読む

### このissueで解決すること
ホスト層フィルタ（`issue-phase001-003`）で各ビューの表示は絞り込めるようになったが、**エディタ本文側には絞り込み手段が無い**。長い管理ノートでは「完了タスクを隠して未完だけ見たい」「`#総務` だけ見たい」といった視界整理ができず、ノイズの中で抜け漏れが起きやすい。

本 issue では、エディタ上で条件にヒットしない（またはヒットする）タスク行を **Excel の行非表示と同じく“折り畳んで表示しないだけ”** にする。Markdown 本文は一切改変しないため、「唯一の正」を壊さずに視界だけ整える（プラン §8.2）。

### 方針
- 既存の構造化フィルタ `filterNodes`（`src/lib/query/`）を**判定にのみ**流用し、対象タスクとその子（`@meta`・`>` メモ・子タスク）の行域集合を算出。
- CodeMirror6 の装飾で対象行範囲を非表示化（`Decoration.replace({block:true})` 相当）。本文テキストは無改変。
- 「ヒットのみ表示／ヒットを隠す」の双方向を切替。トグルで即全展開。

### 仕様（確定事項：迷ったらこれに従う）
- 非表示単位は **タスクノード＋その全子孫の行域**（`@meta` 子リスト・`>` メモ・子タスクを含む）。途中行だけ隠さない。
- フィルタは**表示レイヤのみ**。本文・AST・書き戻しに一切影響しない。
- 方向は「ヒットのみ表示」「ヒットを隠す」の 2 モードを設定で切替。
- 非表示中は「N 件非表示」のプレースホルダ表示／完全 0 高さ のどちらかを選択式（既定はプレースホルダ＝抜け漏れ防止と整合）。
- アンフォールド用のコマンド（全展開）を必ず用意する。
- フィルタ条件は §8.4 の通り、まずは**エディタ独立**（ホスト層フィルタとは別ストア）でよい。共通同期は別 issue（`AE`）。

### 実装の要点・つまずき
- CM6 のブロック非表示は行域がズレると崩れる。行域算出は `nodeLineMap` ＋ AST から**純関数**で出し、テスト可能にする。
- 折り畳み中に本文が編集されたら StateField を再計算（ドキュメント変更に追従）。
- 大きいファイルでの再計算はデバウンス（既存の同期デバウンス設定を流用）。
