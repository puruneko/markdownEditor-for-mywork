# マークダウンスケジュール管理記法 ガイド

> このドキュメントは、タスク・スケジュール管理ファイルをこのプロジェクトの記法に変換する際の参考資料です。
> パーサー実装（`src/lib/parser/md-to-ast.ts`）・シリアライザー実装（`src/lib/parser/ast-to-md.ts`）・デモファイル（`demo/demo_markdown.md`）を正とします。

---

## 1. 全体構造

ドキュメントは **Section（見出し）** → **Node（リスト行）** の2層で構成されます。

```
# セクション見出し（H1〜H6）
> セクション補足（任意）

- グループ（ListNode / TaskNode）
  - [ ] 子タスク（TaskNode）
    - @schedule: 2026-06-01T10:00/2026-06-01T11:00
    - @due: 2026-06-01
    > 補足コメント（QuoteNode）
```

---

## 2. セクション（見出し）

Markdown 標準の `#` 記法をそのまま使います。

| 書き方 | 用途 |
|---|---|
| `# タイトル` | H1：ドキュメント全体のタイトル |
| `## セクション名` | H2：大カテゴリ |
| `### サブセクション名` | H3〜：小カテゴリ |

- セクションは深さに応じて自動的に入れ子になります
- セクション直下に `> 補足` を置くと、そのセクションの説明メモになります

```markdown
## 【清掃業者対応】㈱クリーンプロ 定期清掃

> 月次清掃は毎月第2・第4金曜 8:00〜12:00
> 松田さんへの連絡はできるだけ前日までに。
```

## 3. タスク行（チェックボックス付きリスト）

### 3.1 ステータスマーカー

```
- [ステータス] タスクテキスト
```

| マーカー | ステータス | 意味 |
|---|---|---|
| `- [ ]` | `todo` | 未着手 |
| `- [x]` | `done` | 完了 |
| `- [>]` | `doing` | 進行中・対応中・待ち中 |
| `- [!]` | `blocked` | 要注意・緊急・ブロック |
| `- [-]` | `hold` | 保留・キャンセル・スキップ |

### 3.2 具体例

```markdown
- [ ] 松田さんへ日程変更の電話
- [x] 社内掲示板に清掃日程変更のお知らせ掲示
- [>] 見積書受領 → 6/18 到着予定が未着
- [!] 契約満了日：2026-07-31。7/15までに締結必須
- [-] 社内報の原稿収集 → 今月は休刊のためキャンセル
```

### 3.3 ステータス選択の判断基準

| 状況 | 使うマーカー |
|---|---|
| 未着手・これからやる | `[ ]` |
| 完了した | `[x]` |
| 進行中・相手の回答待ち・手配中 | `[>]` |
| 緊急・絶対に忘れてはいけない・デッドライン付き重要事項 | `[!]` |
| 保留中・今月はキャンセル・スキップ | `[-]` |

## 4. メタ情報（@フィールド）

タスク行の**子リスト**として、`- @キー: 値` の形式で記述します。

```markdown
- [ ] タスクテキスト
  - @schedule: 2026-06-01T10:00/2026-06-01T11:00
  - @due: 2026-06-15
  - @priority: 1
  - @dependsOn: タスクID1, タスクID2
  - @tags: 総務, 清掃
```

### 4.1 @schedule（スケジュール）

カレンダー・ガントチャートへの表示に使われる最重要フィールドです。

**形式：** `YYYY-MM-DDTHH:mm/YYYY-MM-DDTHH:mm`（開始日時/終了日時）

```markdown
- [ ] 清掃立ち会い
  - @schedule: 2026-06-27T08:00/2026-06-27T12:00
```

| パターン | 形式 | 例 |
|---|---|---|
| 時刻付き（同日） | `YYYY-MM-DDTHH:mm/YYYY-MM-DDTHH:mm` | `2026-06-01T10:00/2026-06-01T11:00` |
| 時刻付き（複数日） | `YYYY-MM-DDTHH:mm/YYYY-MM-DDTHH:mm` | `2026-06-01T08:00/2026-06-03T17:00` |
| 日付のみ | `YYYY-MM-DD/YYYY-MM-DD` | `2026-06-01/2026-06-05` |

- 秒は含めない（分単位）
- 同一タスクに複数の `@schedule` を書くことが可能（複数日にまたがる作業など）

```markdown
- [ ] 廃棄対象書類の仕分け作業
  - @schedule: 2026-06-24T14:00/2026-06-24T17:00
  - @schedule: 2026-06-25T14:00/2026-06-25T17:00
  - @due: 2026-06-26
```

### 4.2 @due（期限）

```markdown
- [ ] 稟議書作成
  - @due: 2026-06-30
```

**形式：** `YYYY-MM-DD`（日付のみ）

### 4.3 その他のメタフィールド

| フィールド | 形式 | 例 |
|---|---|---|
| `@priority` | 整数（小さいほど高優先） | `@priority: 1` |
| `@dependsOn` | ID のカンマ区切り | `@dependsOn: id1, id2` |
| `@tags` | 文字列のカンマ区切り | `@tags: 総務, 清掃, 月次` |

## 5. リスト行（チェックボックスなし）

チェックボックスのない `- テキスト` 行は **グループ** または **メモ** として機能します。

### 5.1 グループ（ListNode / isGroup）

子にタスクを含む場合、グループとして扱われます。ガントチャートでは「フェーズ」や「カテゴリ」として表示されます。

```markdown
- 6月定期清掃（第2回）日程変更対応          ← グループ（子にタスクあり）
  - [x] 松田さんへ日程変更の電話
  - [ ] 6/27 清掃立ち会い
```

### 5.2 メモ（ListNode / isMemo）

子にタスクを含まない場合、メモとして扱われます。カレンダー・ガントには表示されません。

```markdown
- 注意事項
  - 来客用お茶は緑茶・ほうじ茶・紅茶の3種
  - 手土産のお返しは原則不要
```

---

## 6. 補足コメント（ブロッククオート）

`> テキスト` 形式で、メモ・補足・状況説明を書きます。パーサーはこれを `QuoteNode`（isMemo: true）として扱い、タスクとしての処理は行いません。

### 6.1 セクション直下のコメント

```markdown
## 【会議室・来客対応】

> 来客用お茶：緑茶・ほうじ茶・紅茶・コーヒー（インスタント）
> 手土産のお返しは原則不要（3,000円以下は受け取り可）
```

### 6.2 タスク・リスト行の子コメント

タスクや `[>]` 行の状況説明・経緯・注意点を書く場合に使います。

```markdown
- [>] 清掃報告書を松田さんからもらう → まだ来ていない
  - @due: 2026-06-20
  > 5月分の報告書も未着。6/27立ち会い時にまとめて受領する予定。
  > 松田さん曰く「本社から郵送手配中」とのこと。
```

---

## 7. インデントルール

**2スペース**単位で子要素をインデントします。

```
- 親リスト行（indent 0）
  - [ ] 子タスク（indent 2）
    - @schedule: ...（indent 4）
    > 子コメント（indent 4）
    - [ ] 孫タスク（indent 4）
      - @due: ...（indent 6）
```

| 要素 | インデント |
|---|---|
| セクション直下のノード | 0 |
| 子ノード | 親 + 2 |
| @metaフィールド | 親タスク + 2 |
| 子コメント（`>`） | 親タスク + 2 |

## 8. ドキュメント全体のパターン

### 8.1 標準パターン

```markdown
# ドキュメントタイトル YYYY年M月

> 担当：氏名
> 更新：YYYY-MM-DD

## セクション1

> セクション補足メモ

- グループ名
  - [ ] タスク1
    - @schedule: 2026-06-01T10:00/2026-06-01T11:00
    - @due: 2026-06-01
  - [x] 完了タスク
    - @schedule: 2026-05-20T14:00/2026-05-20T15:00
  - [>] 進行中タスク → 状況説明
    - @schedule: 2026-06-10T09:00/2026-06-10T09:30
    - @due: 2026-06-15
    > 経緯・注意事項をここに書く

## セクション2

- [!] 緊急タスク
  - @due: 2026-06-20
  > 理由・背景
```

### 8.2 メモ書き・未整理メモのパターン

チェックボックスなしリストと `>` コメントを組み合わせます。

```markdown
## 【未整理メモ】

- [ ] 給湯室の電子レンジを確認する
  > 開発部の田辺さんと経理の宮本さんから「温まらない」と複数の声。
  > メーカー保証期間内かどうか確認（購入日を調べる）。

- [!] 外部業者の入館証管理を見直す
  > 台帳が古くなっている。退職した業者担当者が持ったままの可能性。
```

## 9. 変換ルール：他の記法 → この記法

### 9.1 プレーンテキストのタスクメモ

```
# 変換前（プレーンテキスト）
・清掃業者に電話する（6/5 9時）
・見積もり受領（期限 6/25）→ まだ来ていない
・稟議書作成（6/30まで）
  - 予算コードを経理の佐藤さんに確認してから記入する

# 変換後
- [ ] 清掃業者に電話する
  - @schedule: 2026-06-05T09:00/2026-06-05T09:30
- [>] 見積もり受領 → まだ来ていない
  - @due: 2026-06-25
- [ ] 稟議書作成
  - @due: 2026-06-30
  > 予算コードを経理の佐藤さんに確認してから記入する
```

### 9.2 他のMarkdown タスク記法（Obsidian Tasks など）

```
# 変換前（Obsidian Tasks 記法）
- [ ] 清掃業者に電話する 📅 2026-06-05 ⏰ 09:00
- [x] 社内掲示板に掲示 ✅ 2026-06-05

# 変換後
- [ ] 清掃業者に電話する
  - @schedule: 2026-06-05T09:00/2026-06-05T09:30
- [x] 社内掲示板に掲示
  - @schedule: 2026-06-05T10:00/2026-06-05T10:30
```

### 9.3 箇条書きメモ → グループ構造

```
# 変換前
清掃業者対応
  ・電話済み（6/5）
  ・見積書待ち（期限 6/25）
  ・稟議書これから

# 変換後
- 清掃業者対応
  - [x] 電話
    - @schedule: 2026-06-05T09:00/2026-06-05T09:30
  - [>] 見積書受領待ち
    - @due: 2026-06-25
  - [ ] 稟議書作成
    - @due: 2026-06-30
```

### 9.4 表形式のタスク管理

```
# 変換前（表）
| タスク | 期限 | 状態 |
|---|---|---|
| 清掃業者電話 | 6/5 | 完了 |
| 見積書受領 | 6/25 | 待ち |

# 変換後
- [x] 清掃業者電話
  - @due: 2026-06-05
- [>] 見積書受領
  - @due: 2026-06-25
```

## 10. 変換時の判断フロー

```
タスク・スケジュール情報を得る
  │
  ├─ 日時（時刻）がある → @schedule: YYYY-MM-DDTHH:mm/YYYY-MM-DDTHH:mm
  ├─ 日付（締め切り）がある → @due: YYYY-MM-DD
  │
  ├─ 状態を判定
  │    ├─ 未着手・これからやる → [ ]
  │    ├─ 完了 → [x]
  │    ├─ 進行中・待ち・手配中 → [>]
  │    ├─ 緊急・重要期限あり → [!]
  │    └─ 保留・キャンセル → [-]
  │
  ├─ タスクが複数あり階層がある → リストでグループを作る
  │    例: - グループ名
  │         - [ ] 子タスク
  │
  ├─ メモ・状況説明・経緯がある → > コメントとして子に置く
  │
  └─ セクション（カテゴリ）がある → ## 見出し で区切る
```

## 11. よくある変換ミスと注意点

| NG | OK | 理由 |
|---|---|---|
| `- [ ] タスク @schedule: 2026-06-01T10:00/11:00` | `- [ ] タスク`<br>`  - @schedule: 2026-06-01T10:00/2026-06-01T11:00` | @meta はタスク行内ではなく子リストに書く |
| `- [x] タスク (done)` | `- [x] タスク` | 状態は `[x]` マーカーで示す。テキスト内に状態を重複して書かない |
| `> - [ ] タスク` | `- [ ] タスク` | `>` の中のチェックボックスはタスクとして認識されない |
| インデント 4スペース | インデント 2スペース | パーサーは2スペース単位で階層を判定する |
| `@schedule: 2026-06-01T10:00` | `@schedule: 2026-06-01T10:00/2026-06-01T11:00` | schedule は開始/終了の両方が必要 |
| `@schedule: 2026-06-01 10:00〜11:00` | `@schedule: 2026-06-01T10:00/2026-06-01T11:00` | ISO 8601形式（`T`区切り、`/`で開始・終了を区切る） |
